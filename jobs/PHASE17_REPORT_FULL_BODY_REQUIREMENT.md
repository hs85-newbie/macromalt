# Phase 17 상세 리포트 보강 지시문

## 목적

이번 상세 리포트는 URL 접근 가능 여부와 무관하게, **리포트 자체만으로 검증이 가능해야 한다**.
따라서 샘플 발행 결과는 링크만 나열하지 말고, **발행본 본문 전체를 리포트 안에 포함**해야 한다.

## 지시 사항

상세 리포트에는 샘플 5건 각각에 대해 **발행본 본문 전체**를 포함하라.

## 필수 포함 항목

1. Post ID  
2. 발행 시각  
3. 최종 제목  
4. 최종 H2 / H3 구조  
5. 발행된 글 **본문 전체**  
6. opener H2 문구  
7. opener 첫 2~3문장 별도 발췌  
8. 메인 픽 또는 핵심 섹터가 opener 첫 문장에 포함되었는지 여부  
9. 금지 패턴 미포함 여부  
   - 예: `오늘 이 테마를 보는 이유`  
   - 예: macro recap 시작  
10. 참고 출처 영역 포함 여부  

## 권장 추가 항목

- 최종 HTML 스냅샷  
- 발행 화면 캡처 1장  
- verifier / reviser 최종 판정 로그  
- PARSE_FAILED 관련 런타임 로그 요약  

## 작성 원칙

- URL만 나열하지 말 것  
- 보고서 자체만으로 검증 가능해야 함  
- opener만 부분 발췌하는 수준으로 끝내지 말 것  
- **본문 전체를 반드시 포함**할 것  
- 샘플 5건 모두 동일 형식으로 정리할 것  

## 제외 항목

이번 상세 리포트 범위에서 아래 항목은 제외한다.

- 커밋 해시  
- `main` 푸시 여부  
- Git 상태 전반  

해당 항목은 사용자 별도 검증 대상으로 둔다.

## 판정 목적

이번 보강의 목적은 아래를 리포트 단독으로 확인 가능하게 만드는 것이다.

- Post2 opener 구조 개편이 실제 발행본에 반영되었는가  
- opener 첫 문장이 메인 픽 / 핵심 섹터 중심으로 시작하는가  
- Post1형 거시 배경 재서술이 다시 유입되지 않았는가  
- 본문 전체 흐름상 opener → 메인 픽 → 보조 픽 → 종목 리포트 → 체크포인트가 자연스럽게 연결되는가  
- 참고 출처 구조가 최종 발행본에서도 유지되는가
