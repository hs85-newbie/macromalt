## 1. 배경

Phase 16은 **CONDITIONAL GO**로 종료되었고, 이번 Phase 17의 핵심 잔여 이슈는 아래 두 가지다.

1. **Post2 opener 구조 개편**
   - 첫 H2 섹션명이 아직도 generic한 “오늘 이 테마를 보는 이유” 패턴에 머무른다.
   - 첫 문장이 메인 픽 진입이 아니라 거시 배경 재서술로 시작하는 경향이 남아 있다.
2. **PARSE_FAILED 런타임 검증 강화**
   - 정적 표준화는 끝났지만, 실출력 기준의 런타임 검증과 실패 유형 분류가 더 필요하다.

이번 문서는 위 두 문제만 다루는 **개발 에이전트 실행용 최종 지시문**이다.

---

## 2. 이번 Phase 17의 목적

이번 작업의 목적은 미감 개선이 아니다.
핵심은 다음 두 가지다.

- Post2 opener를 **테마 소개형**이 아니라 **픽 각도 진입형**으로 바꾼다.
- `PARSE_FAILED`를 정적 체크가 아니라 **실행 로그 기반으로 관측 가능**하게 만든다.

## 5. 핵심 문제 정의

### 문제 A — Post2 첫 H2가 generic theme explainer로 고정되는 현상
현재 Post2 구조 5번이 “오늘 이 테마를 보는 이유”로 되어 있어,
모델이 opener를 **시장 설명 / 거시 설명 / 테마 소개**로 소비하는 경향이 남아 있다.

### 문제 B — opener 첫 문장이 macro recap으로 시작하는 현상
현재 Post2는 첫 문장에서 바로 종목/섹터 진입을 하지 않고,
금리/환율/유가/불확실성 등 **시장 전반 설명으로 먼저 들어가는 습관**이 남아 있다.

### 문제 C — PARSE_FAILED는 정적 표준화 이후 런타임 관측이 아직 부족함
실패가 나더라도 어느 단계에서, 어떤 섹션에서, 어떤 구조 오류로 깨졌는지
실행 단위 로그가 충분히 남지 않으면 재현성과 보정 우선순위를 잡기 어렵다.

## 7. Post2 opener H2 생성 규칙

### 허용 방향
- 왜 지금 {메인 픽}인가
- 지금 시장이 {메인 픽}에 묻는 질문
- 이번 변수에서 먼저 봐야 할 {핵심 섹터}
- {메인 픽}을 먼저 봐야 하는 이유
- 오늘 테마보다 먼저 확인할 {핵심 종목}

### 금지 방향
- 오늘 이 테마를 보는 이유
- 오늘 시장을 보는 이유
- 최근 거시 환경을 먼저 보면
- 글로벌 매크로 흐름을 먼저 정리하면
- 이번 시장 변수는
- 최근 변동성이 커지면서

### 구현 원칙
- H2는 generic한 템플릿 문구가 아니라 **픽 각도**를 드러내야 한다.
- 메인 픽 또는 핵심 섹터명이 H2에 직접 들어가는 것이 우선이다.
- 자동 생성 시에도 generic fallback H2를 금지한다.

## 8. opener 첫 문장 강제 규칙

### 필수 규칙
- 첫 문장은 메인 픽 또는 핵심 섹터명을 반드시 포함한다.
- 첫 문장은 거시 배경 재서술로 시작하면 안 된다.
- 첫 문장은 “왜 이 종목/섹터가 지금 민감한가”를 바로 설명해야 한다.
- 첫 문장 안에 투자 판단의 핵심 변수 1개 이상이 들어가야 한다.

### 허용 가능한 핵심 변수 예시
- 수주
- 판가
- 재고
- 운임
- CAPEX
- 정책 수혜
- 실적 민감도
- 가격 전가력
- 규제
- 밸류체인 위치
- 수요 회복
- 주문 backlog
- 업황 스프레드

### 금지 패턴
- 금리/환율/유가/경기 우려 등 macro 변수만 먼저 설명하는 시작
- “불확실성이 커졌다”, “변동성이 확대됐다” 같은 범용 문장 단독 시작
- 첫 문장과 둘째 문장 안에 메인 픽/핵심 섹터명이 모두 빠지는 구성

### 권장 패턴 예시
- {메인 픽}은 이번 변수에서 {핵심 변수} 해석이 먼저 붙는 종목이다.
- 이번 테마에서 먼저 봐야 할 것은 시장 전체보다 {핵심 섹터}의 {핵심 변수}다.
- {메인 픽}은 단순 테마 수혜보다 {실적/수주/가격 전가력/정책 연결}이 핵심이다.
- {핵심 섹터}는 이번 흐름에서 지수 민감도보다 {핵심 변수}로 차별화가 먼저 생긴다.

## 9. Post2 작성 프롬프트 수정 지시

기존 캐시의 픽 작성 프롬프트에 아래 규칙을 추가한다.

### 추가 규칙
- opener는 generic macro recap 섹션이 아니다.
- opener는 메인 픽 또는 핵심 섹터를 기준으로 시작해야 한다.
- opener 첫 문장은 반드시 메인 픽 또는 핵심 섹터명을 포함한다.
- opener 첫 문장은 시장 전체 설명이 아니라 해당 픽/섹터의 핵심 민감도를 먼저 제시한다.
- 거시 배경 설명은 opener 첫 문장 뒤 또는 다음 문단으로 밀어낸다.
- Post1이 이미 제공한 시장 컨텍스트를 Post2 opener가 길게 반복하지 않는다.
- Post2는 “무슨 시장인가”보다 “그래서 어떤 종목/섹터가 먼저 반응하나”를 먼저 보여준다.

### 추가 금지 규칙
- “오늘 이 테마를 보는 이유” 같은 범용 H2 생성 금지
- 첫 문장에서 macro 변수 2개 이상만 나열하는 시작 금지
- opener 첫 2문장 안에 메인 픽/핵심 섹터명 누락 금지
- Post1의 시장 구조 설명을 그대로 복제하는 opener 금지

## 10. verifier / reviser 점검 항목 추가

### 신규 verifier 점검 항목
1. Post2 첫 H2가 generic theme explainer인지
2. opener 첫 문장이 macro recap인지
3. opener 첫 문장에 메인 픽 또는 핵심 섹터명이 포함됐는지
4. opener 첫 2문장이 Post1의 시장 컨텍스트와 역할 중복인지
5. opener가 종목/섹터 민감도 대신 배경 설명만 하고 있는지
6. opener가 뒤이어 나오는 메인 픽 / 보조 픽 / 종목 리포트와 논리적으로 연결되는지

### 신규 reviser 보정 규칙
- generic H2 → pick-angle H2로 교체
- macro-first opener → pick-first opener로 재작성
- 메인 픽/핵심 섹터명 누락 → 첫 문장에 즉시 삽입
- Post1과 중복 → 시장 배경 축소 후 종목 연결 문장으로 치환
- 날짜/수치 보정 과정에서 새 절대 날짜를 발명하지 않는다.

### 검수 출력 형식 추가
- opener 구조 적합 / 부적합
- 문제 문장
- 문제 유형
- 수정 이유
- 교체 문장 제안

## 11. PARSE_FAILED 런타임 검증 강화 지시

이번 Phase에서는 정적 패턴 정리로 종료하지 않는다.
반드시 실제 생성 실행 기준으로 `PARSE_FAILED`를 관측하고 분류한다.

### 최소 실행 요구
- Post2 샘플 실행 최소 5건 이상
- 가능하면 슬롯 2개 이상에서 실행
- 실패가 없더라도 실행 로그는 남긴다.

### 실패 시 반드시 저장할 것
- raw output
- normalized output
- failed section name
- parse stage
- fallback 사용 여부
- 최종 publish blocked 여부

### 최소 실패 분류 체계
- `TYPE_A` — H2/H3 구조 누락 또는 섹션 순서 불일치
- `TYPE_B` — 금지된 opener 패턴으로 인한 후처리 불일치
- `TYPE_C` — HTML 태그 파손 또는 중첩 오류
- `TYPE_D` — source box / 체크포인트 / 종목 구간 파싱 실패
- `TYPE_E` — reviser 수정 후 구조 재파손
- `TYPE_UNKNOWN` — 위 분류로 설명되지 않는 실패

### 필수 로그 필드
- `run_id`
- `slot`
- `post_type`
- `failure_type`
- `parse_stage`
- `failed_section_name`
- `raw_output_snapshot`
- `normalized_output_snapshot`
- `fallback_used`
- `publish_blocked`

- Post2 첫 H2가 아직도 “오늘 이 테마를 보는 이유” 패턴에 머문다.
- Post2 opener 첫 문장이 메인 픽 진입이 아니라 거시 배경 재서술로 시작한다.
- PARSE_FAILED는 정적 정리는 끝났지만 런타임 검증이 부족하다.

이번 작업의 목표:
- Post2 opener를 theme explainer가 아니라 pick-angle opener로 바꾼다.
- opener 첫 문장에서 메인 픽 또는 핵심 섹터를 바로 호출한다.
- PARSE_FAILED를 실행 로그 기준으로 분류 가능하게 만든다.

절대 유지할 것:
- 최근 7일 우선 / 최대 30일 보조 정책 유지
- Post1 = 거시 구조, Post2 = 종목/섹터 민감도 역할 분리 유지
- 기준1(시점 혼합), 기준5(권유성 표현) 유지
- 파라미터 변경은 마지막 수단

실행 지시:

1. Post2 작성 프롬프트 수정
- 출력 구조 5번 “오늘 이 테마를 보는 이유”를 제거한다.
- 5번은 “왜 지금 {메인 픽 또는 핵심 섹터}인가” 성격으로 교체한다.
- generic H2 fallback을 금지한다.

2. opener 첫 문장 규칙 강제
- 첫 문장은 메인 픽 또는 핵심 섹터명을 반드시 포함한다.
- 첫 문장은 macro recap으로 시작하면 안 된다.
- 첫 문장은 해당 픽/섹터의 핵심 민감도를 바로 제시해야 한다.
- 첫 문장에 수주/판가/재고/CAPEX/정책 수혜/실적 민감도/가격 전가력 등 핵심 변수 1개 이상을 넣는다.

3. verifier / reviser 규칙 추가
- Post2 첫 H2가 generic theme explainer인지 검사한다.
- opener 첫 문장이 macro recap인지 검사한다.
- opener 첫 문장에 메인 픽/핵심 섹터명이 있는지 검사한다.
- Post1과 역할 중복이면 pick-angle 문장으로 다시 쓴다.
- 날짜/수치 보정 과정에서 새 절대 날짜를 발명하지 않는다.

4. PARSE_FAILED 런타임 검증 추가
- Post2 샘플 실행 최소 5건 이상 수행한다.
- 실패 시 raw output과 normalized output을 모두 저장한다.
- parse stage, failed section name, fallback_used, publish_blocked를 로그에 남긴다.
- 아래 타입으로 분류한다:
  - TYPE_A: H2/H3 구조 누락 또는 순서 불일치
  - TYPE_B: opener 패턴 불일치
  - TYPE_C: HTML 태그 파손
  - TYPE_D: source/checkpoint/종목 구간 파싱 실패
  - TYPE_E: reviser 수정 후 구조 재파손
  - TYPE_UNKNOWN: 기타

5. 성공 기준
- “오늘 이 테마를 보는 이유” 패턴 0건
- generic opener H2 0건
- opener 첫 문장 macro recap 시작 0건
- opener 첫 문장에 메인 픽 또는 핵심 섹터명 100% 포함
- PARSE_FAILED 발생 시 실패 로그와 유형 분류가 남을 것

6. 보고 형식
- 작업 개요
- 수정한 프롬프트/규칙 요약
- 샘플 실행 결과 요약
- opener 위반 건수
- PARSE_FAILED 건수 및 유형 분포
- 실제 발행 여부와 공개 URL
- 기준1/기준5 판정
- 최종 판정(GO / CONDITIONAL GO / HOLD)

우선순위:
1. opener 구조 강제
2. Post1/Post2 역할 분리 유지
3. PARSE_FAILED 관측 가능성 확보
4. 런타임 재현성 확보
5. 문장 미세 다듬기

적용 후 샘플 실행 결과와 함께 보고하라.
