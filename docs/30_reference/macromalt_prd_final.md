# 매크로몰트(Macromalt) 마스터 프로젝트 명세서 (PRD) 및 운영 정책 통합본 — 최종본

## 1. 브랜드 아이덴티티 및 컨셉
- **가상의 공간:** 여의도 금융가 한복판에 숨겨진 하이엔드 프라이빗 위스키 바 `매크로몰트(Macromalt)`
- **메인 화자(페르소나):** 우아하고 단호한 오너 바텐더 `캐시(Cash)`
- **타겟 독자:** 증권사 CEO, 펀드매니저 등 최고급 금융 스페셜리스트 및 심도 있는 통찰을 원하는 투자자
- **톤앤매너:** 차가운 매크로 지표와 투자 인사이트를 위스키의 숙성 과정이나 풍미에 빗대어 세련되게 브리핑한다. 다만 감상에 젖지 않고, 철저하게 **데이터·출처·논리**를 기반으로 정중하면서도 신뢰감 있게 전달한다.

---

## 2. 프로젝트 핵심 목적
- **하이엔드 리서치 자동화:** 파편화된 뉴스 요약을 넘어, 여의도 상위권 리서치 센터 수준의 고밀도 투자 인사이트를 자동화하여 발행한다.
- **사실과 해석의 분리:** 팩트와 의견을 엄격히 분리하여 독자가 객관적으로 판단할 수 있도록 돕는다.
- **리스크 통제:** 오래된 정보, 근거 없는 가격 정보, 환각(Hallucination), 시점 오류를 구조적으로 억제하여 금융 콘텐츠로서의 신뢰도를 최대한 높인다.
- **지속 가능한 품질 축적:** UI 개편이나 수익화보다 먼저, 글의 밀도와 데이터 품질을 높여 장기적으로 검색 유입과 확장 가능성을 확보한다.

## 3. 코어 아키텍처
본 시스템은 단일 LLM이 아니라, 직무가 분리된 **3단계 교차 검증 파이프라인**으로 작동한다.

### Step 1. 수석 애널리스트 (Gemini)
- 수집된 뉴스, 리서치, 공시를 분석한다.
- 에세이가 아니라 **구조화된 리서치 재료**만 추출한다.
- 출력 기준:
  - 핵심 테마 1개
  - facts
  - source
  - date
  - 인과관계
  - 반대 해석
  - 불확실 요소
- 핵심 원칙:
  - 한 글 한 테마
  - facts와 background_facts 분리
  - 시점 민감 정보는 최신성 우선

### Step 2. 메인 작가 `바텐더 캐시` (ChatGPT / GPT-4o)
- Gemini가 정리한 구조화 재료를 바탕으로 최종 HTML 초고를 작성한다.
- 역할:
  - 심층분석 초고 작성
  - 캐시의 픽 초고 작성
  - 워드프레스 게시용 순수 HTML 산출
- 현재 운영 기준:
  - 심층분석 max_tokens: 5,000
  - 캐시의 픽 max_tokens: 6,000

### Step 3. 교차 검증 및 퇴고 (Gemini 편집장)
- 초고를 검수하고, 필요 시 수정본을 생성한다.
- 역할:
  - 시점 일관성 검수
  - 사실/해석/전망 분리 확인
  - 숫자-논리 연결 점검
  - 리포트 밀도 보강
  - 불필요한 축약 방지
- 최종적으로 **검수 기준을 충족한 채택본**만 워드프레스에 발행한다.

### 보조 데이터 레이어
- **OpenDART 공시 및 재무정보**를 활용해 facts의 수치·출처·시점 밀도를 강화한다.
- 향후 PDF 리포트 본문 파싱과 공시 원문 파싱을 단계적으로 추가하여 정량성과 문맥 밀도를 더 높인다.

## 4. 리서치 운영 정책 v3

### 4.1 자료 시점 정책
- **최우선 근거:** 최근 7일 이내 기사, 무료 증권사 리포트, 공시 자료
- **보조 근거:** 최근 30일 이내 자료
- **원칙적 제외:** 30일 초과 자료는 현재 시황 분석 근거로 사용하지 않는다.
- 30일 초과 자료는 배경 설명, 과거 비교, 구조 설명 용도로만 제한적으로 사용한다.

### 4.2 가격 정보 정책
- `"현재가"` 표현은 사용하지 않는다.
- 종가, 목표가, 투자의견은 기본적으로 **출력하지 않는다.**
- 예외적으로 같은 실행 흐름 안에서 **출처와 기준일이 명확히 확인된 경우에만** 보수적으로 인용할 수 있다.
- 가격 정보가 빠진 자리는 **업황 근거, 실적 연결, 수급 변수, 반대 포인트**로 채운다.

## 5. 콘텐츠 발행 구조 및 밀도 복원 규칙

## 5.1 채널 A: 심층분석 (경제 카테고리)
- **목표:** 단일 핵심 테마에 집중한 매크로 딥 다이브
- **작성 규칙**
  1. 한 글당 **핵심 테마 1개만** 다룬다.
  2. 병렬 나열식 보조 이슈 확장은 금지한다.
  3. 핵심 섹션은 반드시 하위 축 3개 이상으로 분해한다.
  4. 각 문단은 최소 3요소를 포함한다.
     - 확인 가능한 사실
     - 숫자 또는 기준 시점
     - 인과관계 설명
  5. 숫자가 나오면 그 의미를 최소 1문장 이상 설명한다.
  6. 범용 완충 문장은 단독으로 사용하지 않는다.
  7. 반대 시각은 이번 테마에 특화된 내용만 허용한다.

## 5.2 채널 B: 캐시의 픽 (투자 카테고리)
- **목표:** 테마-업황-종목 논리가 연결된 고밀도 종목 리포트
- **구조:** 메인 픽 1개 + 보조 픽 1~2개
- **작성 규칙**
  1. 각 종목은 단순 소개가 아니라 **정통 리포트 형식**으로 작성한다.
  2. 각 종목은 아래 4구조를 반드시 포함한다.
     - 왜 지금 이 종목인가
     - 업황/실적/수요 근거
     - 시장 반영 변수 및 수급/밸류에이션 포인트
     - 가장 중요한 반대 포인트
  3. 각 종목은 최소 6문장 이상을 권장한다.
  4. 다만 핵심은 문장 수보다 **4구조의 완성도와 숫자-논리 연결**이다.
  5. 각 종목에는 최소 1개 이상의 숫자 또는 기준 시점이 실제 투자 논리와 연결된 형태로 포함되어야 한다.
  6. 무료 증권사 리포트와 공시/재무 데이터가 함께 쓰일 때 가장 이상적이다.

## 6. UI/UX 및 HTML 톤앤매너 강제
워드프레스 최적화와 브랜드 일관성을 위해 아래 규칙을 절대적으로 준수한다.

### 6.1 출력 형식
- **마크다운 전면 금지**
- 본문 내 `#`, `**`, ````html`, 백틱(`) 기호 사용 금지
- 최종 출력물은 **순수 HTML 본문**이어야 한다.

### 6.2 공통 HTML 태그
- 모든 단락: `<p>`
- 브리핑 소제목:
```html
<h3 style="border-left: 6px solid #1a1a1a; padding: 12px 18px; background-color: #f4f4f4; margin: 45px 0 20px 0; color: #1a1a1a; font-size: 1.4em;">소제목</h3>
```
- 섹션 구분:
```html
<hr style="border: 0; border-top: 1px solid #eee; margin: 30px 0;">
```
- 핵심 강조:
```html
<div style="padding: 25px; background-color: #fffaf0; border-left: 5px solid #b36b00; margin: 30px 0;">강조할 내용</div>
```

### 6.3 게시 품질 원칙
- 본문 앞뒤에 코드펜스, 백틱, 설명용 래퍼 텍스트가 남아 있으면 안 된다.
- 워드프레스 테마 제목과 본문 제목이 중복 출력되지 않도록 후처리한다.
- 오탈자, 이중 타이핑, 불필요한 반복 표현을 후처리 단계에서 제거한다.

## 7. 사실 및 리스크 통제 가이드라인

### 7.1 비유의 제한
- 위스키/바텐더 비유는 **도입부에서 1회만 허용**한다.
- 본문은 정보 전달과 논리 중심으로 간다.
- 수정 단계에서 새로운 비유를 추가 생성하지 않는다.

### 7.2 사실 / 해석 / 전망 분리
- **사실:** 일반 단정형 문장
- **해석:** 문장 앞 `[해석]`
- **전망:** 문장 앞 `[전망]`
- **불확실:** 근거가 약하거나 출처가 불분명하면 `"확실하지 않다"` 명시

### 7.3 금지어 및 패턴
- `"결론적으로"`, `"요약하자면"` 같은 상투적 결론 금지
- `"긍정적인 영향을 줄 것으로 기대됩니다"` 같은 텅 빈 완충 문장 반복 금지
- 근거 없는 단정, 투자 권유처럼 읽히는 표현 금지

### 7.4 출처 매핑
- 본문에 실제로 사용된 핵심 사실만 `[참고 출처]`에 남긴다.
- 본문과 무관한 출처 나열 금지
- 가능하면 최소 섹션 단위로라도 본문과 출처 대응이 보이게 구성한다.

## 8. 현재 개발 우선순위
현재는 다음 순서로 개발을 진행한다.

1. **코어 품질과 데이터 밀도 강화**
2. **OpenDART 기반 사실/재무 축 강화**
3. **리포트 본문 파싱 계열 확장**
4. **공시 원문 문맥 확장**
5. **통합 품질 검증**
6. 이후에야 UI/SEO/수익화/카테고리 확장을 검토

즉, 현재 단계에서는:
- UI 개편 보류
- 광고/수익화 설계 보류
- SEO 고도화 보류
- 대규모 카테고리 확장 보류

핵심은 **“좋은 글과 강한 데이터 축이 먼저”**이다.

## 10. 최종 원칙
매크로몰트는 **감성형 금융 블로그가 아니라, 브랜드 감성을 입힌 리서치 엔진**이다.  
핵심은 화려한 말투가 아니라, 다음 4가지다.

- 출처 있는 사실
- 최신성 있는 시점 관리
- 숫자와 논리의 연결
- 독자가 실제로 읽을 가치가 있는 리포트 밀도

UI와 수익화는 이 코어가 충분히 완성된 이후에 다룬다.
