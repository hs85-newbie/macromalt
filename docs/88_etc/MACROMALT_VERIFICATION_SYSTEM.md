```
Step 1 (Gemini)  → 분석 재료 생성

         ↓
    [후처리 A] 코드펜스 / 마크다운 자동 제거   _strip_code_fences()
    [후처리 B] 중복 타이핑 교정               _fix_double_typing()

    [검증기 2] Phase 13 해석 지성             _check_interpretation_quality()
    [검증기 3] Phase 14 시제 검증             _check_temporal_sanity()
    [검증기 4] Phase 14 수치 이상값           _check_numeric_sanity()
         ↓
Step 3 (Gemini)  → 팩트체크 25개 기준        verify_draft()
                   (유일한 실제 차단 관문)
         ↓

    [검증기 5] Post 분리도                   _check_post_separation()

    [검증기 7] Phase 19 품질로그 4개 필드    _calc_quality_pass_fields()
```

## 후처리 A — 코드펜스 / 마크다운 자동 제거

**함수:** `_strip_code_fences(text)`
**적용 시점:** Step 2 초고 생성 직후 + Step 3 수정본 수령 직후 (모든 경로 공통)

GPT/Gemini가 마크다운을 출력했을 때 자동으로 제거하는 5단계 처리.

| 단계 | 처리 내용 |
|------|-----------|
| 1 | 앞뒤 코드펜스 제거 — ` ```html `, ` ``` ` |
| 2 | 문서 내 잔여 코드펜스 제거 — 중간에 삽입된 ` ```html ` |
| 3 | 불필요한 따옴표 래퍼 제거 — `"<h1>..."` → `<h1>...` |
| 4 | HTML 앞 설명 텍스트 제거 — `"다음은 HTML입니다:\n<h1>..."` → `<h1>...` |
| 5 | HTML 뒤 설명 텍스트 제거 — 마지막 닫기 태그 이후 200자 미만 잡문 제거 |

**사전 지시 (GPT 프롬프트):**
마크다운이 생성되지 않도록 프롬프트에 아래 블록이 항상 포함된다.

```
[포맷 규칙 — 마크다운 절대 금지]
- 마크다운 기호(#, ##, **, *, `, ---, [ ]) 절대 사용 금지
- 코드 블록(```html 등) 절대 금지. HTML 태그로 바로 시작.
- 허용 태그: <h1>, <h3>, <p>, <hr>, <strong> (인라인 스타일 포함)
```

GPT 출력 오류 자동 교정.

| 입력 패턴 | 교정 결과 |
|-----------|-----------|
| `[해해석]` | `[해석]` |
| `[전전망]` | `[전망]` |
| `[해 해석]` | `[해석]` |
| 한글 단어 즉시 반복 (`미 미`, `을 을`) | 단일로 정리 |
| 연속 공백 | 단일 공백 |

## 검증기 1 — Phase 12 품질 스코어

**함수:** `_score_post_quality(content)`
**적용:** Step 2 완료 직후 / 비파괴 진단 (차단 없음, 로그만 기록)

| 항목 | PASS | WARN | FAIL |
|------|------|------|------|
| 숫자 밀도 | 5개↑ | 2~4개 | 2개 미만 |
| 시점 앵커 | 3개↑ | 1~2개 | 0개 |
| 반론 존재 | 2개↑ | 1개 | 0개 |
| 일반어 남용 | 2개↓ | 3~5개 | 6개↑ |
| 근거 결합 (출처 패턴) | 2개↑ | 1개 | — |

**일반어 16종 (남용 감지 대상):**
`긍정적`, `영향을 줄 수 있다`, `주목할 필요`, `관심이 필요`,
`부각될 수 있다`, `기대감`, `수혜 기대`, `기대감이 높아`, `전망이 밝`,
`불확실성이 높아`, `변동성 확대`, `면밀한 모니터링`, `추이를 지켜볼`,
`관심이 집중`, `주목받고 있`, `시장의 관심`

**반론 마커 20종:**
`반대`, `반론`, `다만`, `그러나`, `하지만`, `단,`, `단 `,
`리스크`, `불확실`, `우려`, `하락 가능`, `약세 전환`, `체크포인트`,
`과도한 기대`, `반영 과잉`, `되돌림`, `조정 가능`, `역풍`

## 검증기 3 — Phase 14 시제 검증

**함수:** `_check_temporal_sanity(content)`
**기준:** 미래 사실을 현재로 단정하는 표현 금지

| 판정 | 조건 |
|------|------|
| FAIL | `FUTURE_AS_FACT` 플래그 1개↑ |
| WARN | 시제 경고 있으나 FUTURE_AS_FACT 없음 |
| PASS | 이상 없음 |

## 검증기 4 — Phase 14 수치 이상값

**함수:** `_check_numeric_sanity(content)`
**기준:** 실현 불가능한 범위의 수치 차단

| 판정 | 조건 |
|------|------|
| FAIL | HARD_FAIL 1개↑ 또는 SUSPICIOUS 2건↑ |
| WARN | SUSPICIOUS 1건 |
| PASS | 이상 없음 |

HARD_FAIL 대상: USD/KRW, 금리(%) 등 정의된 범위를 명백히 벗어난 수치

## 검증기 5 (Step 3) — Gemini 팩트체크 25개 기준

**함수:** `verify_draft(draft, slot, post_type)`
**실제 차단 관문 — pass=false 시 수정본 생성 후 재발행**

### Step 3 판정 유형

| status | 설명 |
|--------|------|
| `PASS` | 팩트체크 이슈 없음, 초안 원본 발행 |
| `REVISED` | 이슈 발견 → Step 3b 수정본 채택 발행 |
| `FAILED_NO_REVISION` | 수정 API 실패(503/timeout) → 초안 원본 발행 |
| `PARSE_FAILED` | Gemini 응답 JSON 파싱 불가 → 검수 skip, 초안 원본 발행 |

### ① 필수 통과 5개 — 위반 시 즉시 pass=false

| # | 기준 | 내용 |
|---|------|------|
| 1 | 시점 일관성 | 최근 7일 중심 구성 / background_facts 현재 시황 근거 사용 금지 / `전망·추정·상향` 포함 문장에 `[전망]` 태그 필수 (확정 과거 실적 제외) |
| 2 | 수치 정확성 | 수치·날짜·고유명사 오류 및 기준 시점 누락 |
| 3 | 사실/해석 구분 | `[해석]`/`[전망]` 태그 없이 해석·전망을 사실처럼 서술 |
| 4 | 출처 단정 | `~할 것이다`, `~가 확실하다` 등 출처 없는 단정 표현 |
| 5 | 투자 권유 | `매수`, `유망`, `담아야`, `기회다`, `추천`, `매력적인 선택지`, `저평가 매력`, `하방 경직성 제공`, `수혜를 받을 수 있는` 등 22종 |

### ② 주제 집중도 6개 — 심각하면 pass=false

| # | 기준 | 내용 |
|---|------|------|
| 6 | 단일 테마 | 핵심 테마 1개에 집중 — 2개 이상 테마 병렬 전개 금지 |
| 7 | 테마 정렬 | 제목·도입부·핵심 섹션 3개가 모두 동일 핵심 테마 |
| 8 | 문단 반복 | 같은 사실을 표현만 바꿔 2회↑ 반복 금지 |
| 9 | 완충 문장 과잉 | `긍정적이다`, `수혜가 예상된다` 류 근거 없이 단독 2회↑ 금지 |
| 10 | 숫자 없는 문단 연속 | 수치 또는 기준 시점 없는 문단 2개↑ 연속 금지 |

### ③ 품질 기준 9개 — 경미하면 issues만, 심각하면 pass=false

| # | 기준 | 내용 |
|---|------|------|
| 11 | 브랜드 톤 | 바텐더/위스키 비유 도입부 1회 초과 금지 |
| **12** | **포맷** | **마크다운 기호(`#`, `##`, `**`, `*`, ` ``` `) 포함 여부** |
| 13 | 문단 밀도 | 핵심 섹션 문단당 사실+숫자+인과관계+영향 4요소 충족 여부 |
| 14 | 숫자 연결 | 숫자가 단순 삽입이 아닌 논리에 연결되어 의미 설명 여부 |
| 15 | 종목 리포트형 | 카드형 요약 축소 금지 — 각 종목 4문장↑, 4단계 순서 준수 |
| 16 | 메인/보조 픽 구분 | 캐시의 픽 메인/보조 픽 역할 차이 명확 여부 |
| 17 | 반대 논리 특화 | 범용 리스크 문장 금지 — 이번 테마 특화 내용 |
| 18 | 캐시 톤 | 캐시 바텐더 톤 과장/부자연 여부 |
| 19 | 출처 대응 | 본문 인용 내용과 참고 출처 직접 대응 여부 |

## 검증기 8 — Phase 19 품질로그 4개 필드

**함수:** `_calc_quality_pass_fields(content)`
**적용:** 정상 발행 완료 후 + PARSE_FAILED 발생 시 운영 로그에 기록
**단일 진실 소스:** 두 경로 모두 이 함수를 공유해 동일 기준 유지

| 필드 | 판정 기준 |
|------|-----------|
| `opener_pass` | pick-angle 패턴(`왜 지금 X인가`) 존재 + 금지 opener 6종 없음 — **Post2 전용** (Post1은 항상 False) |
| `criteria_1_pass` | `지난해` / `작년` / `전년동기` 단독 등장 없음 (시점 혼합 금지 간이 지표) |
| `criteria_5_pass` | `매수`, `유망`, `담아야`, `사야`, `투자 추천`, `강력 추천` 미포함 |
| `source_structure_pass` | `참고 출처`, `📊`, `증권사 리서치`, `뉴스 기사` 중 1개↑ 존재 |
