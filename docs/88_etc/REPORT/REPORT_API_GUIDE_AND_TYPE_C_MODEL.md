## 1. 사용자 API 발급 가이드

> BYOK(Bring Your Own Key) 모델: 사용자가 직접 API 키를 발급받아 macromalt에 입력

---

### 1-1. OpenAI API 키 발급

**소요 시간**: 약 5~10분

**비용**: 사용량 기반 후불 (선불 크레딧도 가능)

#### 발급 절차

```
1단계: 회원가입
  → https://platform.openai.com/signup
  → 이메일 또는 Google 계정으로 가입

2단계: 결제 수단 등록
  → https://platform.openai.com/account/billing
  → 신용카드 등록 (해외 결제 가능 카드 필요)
  → 최소 $5 선불 충전 또는 후불 설정

3단계: API 키 생성
  → https://platform.openai.com/api-keys
  → "+ Create new secret key" 클릭
  → 이름 입력 (예: "macromalt")
  → 생성된 키 복사 (sk-proj-로 시작, 1회만 표시됨)

4단계: 월 한도 설정 (권장)
  → https://platform.openai.com/account/limits
  → "Monthly budget" → $20~$60 설정
  → 한도 초과 시 자동 차단 (과금 방지)
```

#### 예상 비용

| 사용 패턴 | 월 비용 (GPT-4o 기준) |
|---------|-------------------|
| 글 5개/일 (월 150개) | $15~20 |
| 글 21개/일 (월 630개) | $45~55 |

#### 주의사항

- API 키는 절대 공개하지 말 것 (코드, GitHub, 채팅 금지)
- 미사용 시 키 삭제 또는 비활성화 권장
- 프로젝트별로 별도 키 발급 권장 (권한 분리)

## 2. API 발급 대행 가능 여부

### 2-1. 현황 분석

| API | 대행 가능 여부 | 이유 |
|-----|-------------|------|
| **OpenAI** | ❌ **불가 (약관 위반)** | TOS 4.1조: API 재판매 금지 |
| **Google Gemini** | ❌ **불가 (약관 위반)** | Google Cloud TOS: API 재판매 금지 |
| **OpenDART** | ❌ 불필요 | 무료 + 개인별 발급 가능 |
| **FRED/BOK** | ❌ 불필요 | 무료 + 즉시 발급 |

### 2-2. OpenAI TOS 핵심 조항

> **"You may not use our API to create, sell, or provide products or services whose primary purpose is to provide access to our API."**
> (API 자체에 대한 접근을 판매하거나 제공하는 것이 주된 목적인 제품을 만들 수 없음)

**하지만 이것은 허용됨**:
- ✅ 우리의 API 키로 콘텐츠 생성 서비스를 운영하고 그 서비스 이용료를 받는 것
- ✅ BYOK: 사용자 키를 입력받아 사용자의 비용으로 실행
- ✅ 사용량 기반 크레딧(우리가 API 비용 부담, 서비스 이용료 청구)

**핵심 구분**: "API 재판매" vs "API를 활용한 서비스 판매"
→ macromalt는 **서비스(콘텐츠 자동 생성)**를 판매하는 것이므로 합법.

### 2-3. 실질적 대안

```
❌ 직접 대행: "제가 API 키 발급해드릴게요" → 불가
✅ 가이드 제공: "이렇게 발급하세요" 온보딩 가이드 → 가능
✅ 크레딧 모델: 우리 키 사용, 이용료 청구 → 가능
✅ BYOK + 가이드: 사용자 발급, 가이드 제공 → 가장 안전
```

## 3. Type C 크레딧 충전 모델 분석

### 3-1. 모델 개요

> **사용자가 macromalt 크레딧(포인트)을 충전하면, macromalt가 API 비용을 부담하고 글을 생성해주는 방식**

```
[사용자]                    [macromalt]                 [OpenAI/Gemini]
    │                           │                             │
    ├── ₩10,000 충전 ──────────→│                             │
    │                           │                             │
    ├── "반도체 분석글 요청" ────→│── API 호출 ────────────────→│
    │                           │                             │
    │←─ 완성된 글 수신 ──────────│←── 응답 반환 ───────────────│
    │                           │
    │                           └── 크레딧 차감 (-1크레딧)
```

### 3-2. 원가 계산 (Phase 22 기준)

**글 1개 생성 비용** (Post1 기준):

| 항목 | 토큰 수 | 단가 | 비용 |
|------|--------|------|------|
| GPT-4o 입력 | 3,500 | $2.50/1M | $0.00875 |
| GPT-4o 캐시 입력 | 1,000 | $0.625/1M | $0.000625 |
| GPT-4o 출력 | 2,000 | $10.00/1M | $0.020 |
| Gemini 입력(분석+검수) | 4,000 | $0.075/1M | $0.0003 |
| Gemini 출력 | 1,000 | $0.30/1M | $0.0003 |
| **합계** | | | **$0.030 ≈ ₩41** |

> ※ 캐시 히트 70% 가정. 실제 Phase 22 전체 런 기준: 7포스팅 = ~$0.21 = ₩290/런

**글 1개 원가: ₩40~70** (캐시 반영 시)

### 3-3. 크레딧 패키지 설계 (안)

| 패키지 | 가격 | 크레딧 | 글 수 | 글당 단가 | 마진율 |
|--------|------|--------|------|---------|--------|
| **스타터** | ₩10,000 | 200 | **20개** | ₩500/개 | **88%** |
| **베이직** | ₩30,000 | 700 | **70개** | ₩428/개 | **90%** |
| **프로** | ₩100,000 | 2,500 | **250개** | ₩400/개 | **90%** |
| **팀** | ₩300,000 | 8,000 | **800개** | ₩375/개 | **91%** |

> ✅ 원가 ₩40~70 기준, 판매가 ₩375~500 → **마진율 88~92%**

### 3-4. 크레딧 1개 = 글 1개 기준 설정

```
1 크레딧 = 글 1개 생성
- Post1 (분석글): 1크레딧
- Post2 (종목픽): 1크레딧
- Post1 + Post2 세트: 1.5크레딧 (15% 할인 적용)

20크레딧 패키지 = ₩10,000:
  → 분석글 20개 or 세트 13회 발행 가능
  → "한달 매일 글 쓰고 싶으면 ₩30,000 (70개)" 직관적 설명 가능
```

### 3-5. 사용자 관점 가이드 문구 (예시)

```
📦 크레딧 충전 안내

✅ ₩10,000 = 20개 글 생성
   (평균 하루 1개 × 20일 분량)

✅ ₩30,000 = 70개 글 생성
   (매일 2개 × 35일 분량, 3,000원 보너스)

✅ ₩100,000 = 250개 글 생성
   (하루 3슬롯 × 83일 분량, 10,000원 보너스)

💡 글 1개 생성 = 뉴스 수집 + AI 분석 + WordPress 발행 완전 자동화
💡 API 키 불필요 — 키 발급, 비용 관리 모두 저희가 처리
```

### 3-6. Type C 모델의 장단점

#### 장점

| 항목 | 설명 |
|------|------|
| **접근성** | API 발급 없이 즉시 사용 — 진입 장벽 최소 |
| **가격 직관성** | "₩10,000 = 20개 글" → 명확한 가치 제안 |
| **소액 결제** | 구독 부담 없음, 필요할 때만 충전 |
| **마진율 우수** | 88~92% 마진 → 구독 모델(70~80%)보다 높음 |
| **비용 예측** | 운영자 입장에서 크레딧 소비 = 수입 확정 |

#### 단점

| 항목 | 설명 |
|------|------|
| **API 비용 선부담** | 크레딧 미사용 시 비용 없으나, 잔여 크레딧 부채 존재 |
| **API 가격 변동 리스크** | OpenAI 단가 오르면 마진 감소 → 크레딧 단가 조정 필요 |
| **결제 인프라 필요** | 충전 결제 시스템 구현 필요 (Toss Payments 연동) |
| **법적 이슈** | 크레딧 유효기간 설정 시 소비자보호법 고려 |

## 4. BYOK vs Type C 비교 및 권장

### 4-1. 모델 비교표

| 항목 | BYOK (Type B) | 크레딧 (Type C) |
|------|--------------|----------------|
| **사용자 진입 장벽** | 높음 (API 발급 필수) | 낮음 (즉시 결제) |
| **운영자 API 비용** | 없음 (사용자 부담) | 있음 (운영자 부담) |
| **마진 구조** | 구독료 100% = 순이익 | 크레딧 판매액 - API 비용 |
| **타겟 사용자** | 개발자, IT 친숙 사용자 | 일반 투자자, 블로거 |
| **가격 민감도** | API 비용 직접 체감 → 민감 | API 비용 숨겨짐 → 둔감 |
| **구현 복잡도** | 낮음 (키 입력 UI만) | 높음 (결제+크레딧 관리 시스템) |
| **법적 리스크** | 낮음 | 크레딧 유효기간 설정 시 주의 |

### 4-2. 단계적 전환 전략 (권장)

```
Phase 24 출시 초기 (MVP):
  → BYOK 모델로 시작
  → API 발급 가이드 in-app 제공
  → 이유: 빠른 출시, 구현 단순, 초기 사용자 니즈 파악

Phase 25~26 (성장기):
  → Type C 크레딧 모델 추가
  → BYOK와 병행 제공 (사용자 선택)
  → 이유: 비개발자 타겟 확대, 추가 수익원

장기:
  → Type B (구독) + Type C (크레딧) 혼합 제공
  → Starter ₩49,900/월 = 무제한 또는 대량 크레딧 선포함
```
