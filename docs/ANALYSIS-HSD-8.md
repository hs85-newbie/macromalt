## 2. 발견 이슈 목록

### 이슈 #1 — 핵심 버그: HTML 태그 제거 시 개행 손실 (P1)

- **파일**: `generator.py`
- **함수**: `_format_source_section()` → `_rebuild()`
- **라인**: 4761

```python
# ❌ 현재 코드: 모든 HTML 태그를 공백 하나로 치환
raw = re.sub(r"<[^>]+>", " ", match.group(0))
```

`<br>`, `</li>`, `</p>` 등 **줄 구분 역할의 블록 태그**가 공백으로 치환됨.
GPT가 아이템별로 `<br>` 또는 `<li>` 태그를 삽입해도 모두 병합되어 한 문자열이 됨.

**예시:**
```
GPT 원본: <p>매일경제, 2026.04.08<br>연합뉴스, 2026.04.08</p>
태그 제거 후: " 매일경제, 2026.04.08 연합뉴스, 2026.04.08 "
```

### 이슈 #2 — 핵심 버그: 쉼표 구분자가 날짜를 분리 (P1)

- **파일**: `generator.py`
- **함수**: `_format_source_section()` → `_rebuild()`
- **라인**: 4762

```python
# ❌ 현재 코드: 쉼표를 아이템 구분자로 사용
parts = re.split(r"[\n,·•]+", raw)
```

GPT가 `출처명, 날짜` 형식으로 한 아이템을 표현할 때, 쉼표가 아이템 구분자로
처리되어 **출처명과 날짜가 별도 아이템으로 쪼개짐**.

**예시:**
```
입력: "매일경제, 2026.04.08"
분리 결과: ["매일경제", "2026.04.08"]

분류:
  "매일경제" → NEWS_KW("경제") 매칭 → 📰 뉴스 기사 섹션
  "2026.04.08" → 키워드 없음     → 📌 기타 섹션
```

GPT는 자유롭게 `"출처, 날짜"`, `"출처 (날짜)"`, `"날짜 - 출처"` 등 다양한 형식으로
출력할 수 있어 파서 동작이 비결정적임.

## 3. 근본 원인 요약

```
GPT 출력: "매일경제, 2026.04.08<br>연합뉴스, 2026.04.08"
          ↓
_format_source_section() 정규식 매칭 (참고 출처 헤딩 감지)
          ↓
_rebuild() 호출
          ↓
re.sub(r"<[^>]+>", " ", ...) → <br>가 공백으로 치환 → 개행 손실
          ↓
re.split(r"[\n,·•]+", raw) → 쉼표로 분리 → "매일경제" + "2026.04.08" 분리
          ↓
분류 로직: "매일경제" → NEWS_KW → 뉴스 기사
           "2026.04.08" → 미매칭  → 기타
          ↓
출력: 📰 뉴스 기사에 출처명, 📌 기타에 날짜 → 분리 표시
```

## 4. 개선 제안

### 제안 A — `_rebuild()` 파서 수정 (이슈 #1, #2, #3 해결) [필수]

**파일**: `generator.py` — `_format_source_section()._rebuild()` (라인 4760~4787)

```python
def _rebuild(match: re.Match) -> str:
    raw = match.group(0)

    # ✅ 수정 1: 블록 태그를 공백 대신 개행으로 치환 (줄 경계 보존)
    raw = re.sub(r"<br\s*/?>", "\n", raw, flags=re.IGNORECASE)
    raw = re.sub(r"</li>|</p>|</div>", "\n", raw, flags=re.IGNORECASE)
    # ✅ 수정 2: 나머지 HTML 태그만 제거 (내용 없는 태그)
    raw = re.sub(r"<[^>]+>", "", raw)

    # ✅ 수정 3: 쉼표를 구분자에서 제거 → "출처, 날짜" 한 아이템으로 유지
    parts = re.split(r"[\n·•]+", raw)
    items = [p.strip(" \t\r-") for p in parts if len(p.strip()) > 2]
    items = [i for i in items if i.lower() not in ("참고 출처", "참고출처")]
    # (이하 분류/출력 로직 동일)
```

**예상 결과:**
```
입력:  <p>매일경제, 2026.04.08<br>연합뉴스, 2026.04.08</p>
처리:  개행 치환 → "매일경제, 2026.04.08\n연합뉴스, 2026.04.08"
분리:  ["매일경제, 2026.04.08", "연합뉴스, 2026.04.08"]
분류:  둘 다 NEWS_KW("경제") → 📰 뉴스 기사
출력:  📰 뉴스 기사에 "매일경제, 2026.04.08" 한 항목으로 표시 ✓
```

## 6. 검증 기준 (수정 완료 후)

```python
# 테스트 케이스 (단위 테스트)
input_html = """
<h3>참고 출처</h3>
<p>매일경제, 2026.04.08<br>
한국경제, 2026.04.08<br>
KB증권 리서치, 2026.04.07</p>

result = _format_source_section(input_html)

# 검증

assert re.search(r"<li[^>]*>매일경제, 2026\.04\.08</li>", result)
```
