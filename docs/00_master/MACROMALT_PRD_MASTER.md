## 1. 프로젝트 개요

macromalt는 금융·거시 브리핑 블로그 자동화 프로젝트다.  
핵심 산출물은 아래 2종이다.

- **Post1**: 심층 경제/시장 분석
- **Post2**: 종목/섹터 중심의 “캐시의 픽”

핵심 목적은 아래와 같다.

- 출처 있는 사실 기반의 자동 생성
- 최신 시점 관리
- 숫자와 논리의 자연스러운 연결
- 실제 발행 가능한 완성도 확보
- 운영 기준으로 품질을 반복 검증할 수 있는 구조 확보

---

## 2. 핵심 원칙

### 2.1 글 역할 분리
- **Post1**은 거시·시장 구조·파급 경로 중심
- **Post2**는 종목·섹터·트리거·민감도 중심

### 2.2 서술 원칙
- 사실 / 해석 / 전망을 분리한다
- 근거 없는 날짜·수치 보강 금지
- 과거 사실 + 미래 전망을 한 문장에 혼합하지 않는다
- 투자 권유성 표현은 억제한다

### 2.3 검증 원칙
- 초안 기준이 아니라 **실제 발행 + 공개 URL** 기준으로만 품질 검증
- 기준1(시점 혼합)과 기준5(권유성 표현)을 핵심 검수 기준으로 둔다

### 2.4 운영 원칙
- 신규 기능 추가보다 운영 검증을 우선한다
- 파라미터 변경은 최후 수단으로 보고, 먼저 프롬프트/검수 규칙을 보정한다
- 작은 규칙 보정 → 실제 발행 → 공개 URL 검증 순서를 유지한다

## 3. 데이터 사용 우선순위

### 3.1 최우선 근거
- 최근 7일 이내 기사
- 무료 증권사 리포트
- 공시 자료(DART 등)

### 3.2 보조 근거
- 최근 30일 이내 자료

### 3.3 제한
- 30일 초과 자료는 현재 시황의 직접 근거로 사용하지 않는다
- 필요 시 배경 설명 / 과거 비교 / 구조 설명 용도로만 제한 사용한다

## 5. 현재 LLM 파이프라인 개요

### 5.1 기본 구조
- **Gemini**: 분석 재료 구성, 검수, 수정
- **OpenAI**: 본문 작성

### 5.2 파이프라인 단계
1. 뉴스 수집
2. 리서치 수집
3. 분석 재료 생성
4. Post1 작성
5. Post2 작성
6. 검수
7. 수정
8. WordPress 발행

### 5.3 관측
- `DEBUG_LLM=1`로 실제 payload 로그 확인 가능
- 파라미터 변경 없이 먼저 규칙 보정 우선
