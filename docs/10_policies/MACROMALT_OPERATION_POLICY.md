# MACROMALT OPERATION POLICY

기준일: 2026-03-16

## 1. 운영 검증 기준

macromalt는 초안이 아니라 **실제 발행본** 기준으로 검증한다.

필수 조건:
- 실제 생성
- 실제 발행
- 공개 URL 확보
- 공개 URL 기준 품질 판정

다음은 완료로 인정하지 않는다.
- 초안만 있는 상태
- wp-admin 편집 URL만 있는 상태
- 공개 URL 없이 “대충 잘 된 것 같다” 수준의 보고

---

## 2. 핵심 검수 기준

### 기준1 — 시점 혼합
아래를 금지한다.
- 과거 사실을 현재 시황처럼 서술
- 미래 전망을 현재 사실처럼 단정
- 과거 사실 + 미래 전망을 한 문장에 혼합
- 본문에 없는 절대 날짜를 새로 발명하여 삽입

### 기준5 — 권유성 표현
아래를 금지 또는 치환 대상으로 본다.
- 매수 기회
- 저평가 매력
- 하방 경직성 제공
- 수혜를 받을 수 있는
- 실적 개선 효과를 기대할 수 있는
- 기타 직접적 또는 완곡한 투자 권유성 표현

---

## 3. 품질 판정 방식

### GO
- 핵심 검수 기준이 안정적으로 통과
- 실제 발행 및 공개 URL 확보
- 반복 실행에서 품질이 유지됨

### CONDITIONAL GO
- 발행은 가능하나, 운영 모니터링이 필요한 소수 이슈 존재
- 기능보다는 규칙/표현 사전 수준의 잔여 보완 필요

### HOLD
- 발행 품질에 직접 영향 주는 구조적 문제 존재
- 기준1/기준5 반복 위반
- 공개 URL 확보 실패 또는 발행 실패

---

## 4. 보고 형식 원칙

최종 보고에는 아래가 반드시 포함되어야 한다.
1. 작업 개요
2. 사용 입력 요약
3. 실제 생성된 최종 본문 또는 핵심 검증 내용
4. 공개 URL
5. 기준1/기준5 판정
6. source/date 반영 여부
7. 숫자-논리 연결 여부
8. 최종 판정

---

## 5. 운영 관찰 항목

아래는 완전한 실패가 아니라 운영 관찰 항목으로 둔다.
- 반복 주제 선정
- verifier의 기준5 오탐
- market-dominant theme 지속 시 동일 테마 반복

단, 아래는 즉시 보완 대상이다.
- 날짜 hallucination
- 본문에 없는 수치/날짜 삽입
- 공개 발행 URL 누락
- 권유성 표현 재등장

---

## 6. 수정 우선순위

1. 프롬프트/규칙 보정
2. verifier/reviser 규칙 보정
3. 표현 사전 추가
4. 문장 분리 규칙 추가
5. 파라미터 변경은 마지막에 검토

---

## 7. 운영 로그 원칙

운영 로그에는 최소한 아래를 남긴다.
- run_id
- slot
- freshness summary
- 반복 감지 여부
- 발행 성공/실패
- 공개 URL
- 검수 기준 위반 수

---

## 8. 예외 처리 원칙

- Post1 실패 시 Post2 중단 가능
- Post2 실패 시 Post1 결과 유지 후 경고
- date_tier가 `unknown`인 자료는 직접 근거가 아니라 보조 맥락으로 제한
- 30일 초과 자료는 현재 시황 직접 근거로 사용 금지

이 문서는 macromalt의 발행/검증 운영 룰북이다.
