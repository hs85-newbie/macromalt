# MACROMALT DATA POLICY

기준일: 2026-03-16

## 1. 최신성 정책

### 최우선 근거
- 최근 **7일 이내** 자료

### 보조 근거
- 최근 **30일 이내** 자료

### 제한
- **30일 초과** 자료는 현재 시황 / 주제 선정의 직접 근거로 사용하지 않는다
- 필요 시 배경 설명 / 과거 비교 / 구조 설명 용도로만 제한 사용한다

---

## 2. date_tier 정의

모든 수집 자료는 가능한 경우 아래 중 하나로 분류한다.

- `recent`: 최근 7일 이내
- `extended`: 8일~30일 이내
- `old`: 31일 초과
- `unknown`: 날짜 불명

---

## 3. 자료 유형별 날짜 판정 필드

| 자료 유형 | 날짜 필드 | 비고 |
|---|---|---|
| RSS 뉴스 | `published` | feedparser published 기준 |
| Web 수집 | 없음 | `date_tier = "unknown"` 명시 |
| 리서치(네이버/한경) | `published` 또는 수집된 날짜 필드 | source 구현 기준 |
| DART 공시 | `rcept_dt` | 접수일 기준 |
| 사업보고서/공시 원문 | 관련 공시 접수일 | DART 연동 흐름 기준 |

---

## 4. 처리 원칙

### `recent`
- 주제 선정의 최우선 근거
- 핵심 facts로 사용 가능

### `extended`
- 보조 근거
- 핵심 주장보다 맥락 보강용
- 7일 이내 자료가 뒷받침되지 않으면 주제 선정 우선순위 낮음

### `old`
- 수집 단계에서 제외 또는 주제 선정 입력에서 제외
- 필요 시 배경 설명으로만 제한

### `unknown`
- 핵심 숫자 근거로 사용 금지
- auxiliary_context 또는 테마 힌트 수준으로만 사용
- 로그에 건수 남김

---

## 5. 현재 구현/운영 원칙

- `fetch_rss()`에서 `old`는 드롭
- 리서치 수집에도 최신성 필터 적용
- `fetch_web()`에는 `date_tier = "unknown"` 명시
- Gemini 분석 프롬프트에 최신성 우선순위 지시 삽입
- 주제 선정은 최근 7일 자료 중심으로 이루어지도록 유도

---

## 6. freshness 로그

운영 로그에 최소한 아래를 남긴다.

예시:
- 최근 7일 자료 건수
- 8~30일 자료 건수
- 제외(old) 건수
- 날짜불명(unknown) 건수

예시 포맷:
`[Phase 10] 자료 분류: 7일이내 12건 | 8-30일 5건 | 제외(old) 3건 | 날짜불명 4건`

---

## 7. 사용 금지/제한 사례

다음은 금지 또는 제한한다.
- 30일 초과 자료를 현재 시황의 직접 근거로 인용
- 날짜가 없는 자료만으로 주제를 단정
- 오래된 리포트를 최신 시황 판단 근거로 과도하게 사용

---

## 8. 예외
정말 필요한 경우 30일 초과 자료를 쓸 수 있으나,
- 배경 설명
- 구조 설명
- 과거 비교
로만 제한한다.

그 경우에도 본문 핵심 주장의 직접 근거가 되면 안 된다.

이 문서는 macromalt의 최신성/자료 사용 정책 문서다.
