## 1. 현황 요약

### 1.1 배경

2026-03-27 커밋 `0cdc8db` (`refactor(generator): [전망]/[해석] 텍스트 태그 제거 → 자연어 표지로 전환`)에서  
GPT 라이터 프롬프트의 태그 삽입 규칙을 자연어 표지 규칙으로 교체했다.

그러나 리팩토링이 **GPT 라이터 프롬프트만** 갱신하고  
Gemini Reviser / Verifier / 품질 게이트 / 후처리 코드에는 반영하지 않아  
**Gemini 수정 단계에서 태그가 재주입**되는 순환 구조가 지속되고 있다.

### 1.2 태그 생애주기 (현재 파이프라인)

```
GPT Writer  →  [해석]/[전망] 미삽입 (리팩토링 반영)
     ↓
Gemini Verifier  →  품질 게이트 #6: "[해석]" or "[전망]" 존재 여부 체크
                    → 없으면 FAIL (7941-7942행)
     ↓
Gemini Reviser  →  "누락된 [해석]/[전망] 태그 추가" 규칙 실행
                   → 태그 재주입 (5745, 5751행)
     ↓
Phase 15D/E/F  →  **잘못된 시제의 [전망]만** 제거
                  → 해석/전망 태그 전체 제거 로직 없음
     ↓
WordPress 발행  →  [해석]/[전망] 태그 그대로 노출
```

## 2. 발견 이슈 목록

### 이슈 #1 (P0): Gemini Reviser가 태그를 적극 재주입 — 핵심 원인

- **파일**: `generator.py`
- **위치**: `GEMINI_REVISER_SYSTEM` (5745, 5751, 5754, 5758행)
- **내용**:
  ```
  - [해석] / [전망] 태그 누락된 해석·전망 문장에 태그 추가 (삭제 금지, 추가만)
  ...
  1. [해석]/[전망] 태그 누락된 해석·전망 문장에 태그 추가
  ...
  "[전망] 2026년 X는 Y로 예상된다(출처 추정)" 형태로 재서술
  ✅ 수정 후: "[전망] 삼성전자의 2026년 매출은 300조 원으로 추정된다(미래에셋 리서치)"
  ```
- **문제**: GPT가 태그를 생성하지 않아도 Gemini Reviser가 강제로 다시 추가
- **수정 방향**: 해당 4개 항목 → 조건부 언어 표지 규칙으로 교체  
  (예: "미래 예측 문장에 '~로 예상된다' / '~할 것으로 전망된다' 표지가 없으면 추가")

### 이슈 #2 (P0): 품질 게이트가 태그 존재를 합격 조건으로 요구

- **파일**: `generator.py`
- **위치**: 7941~7942행 (내부 품질 체크 `criteria` 딕셔너리)
- **내용**:
  ```python
  " 6. [해석]·[전망]·'확실하지 않다' 규칙이 적용됐는가?":
      "[해석]" in content or "[전망]" in content or "확실하지 않" in content,
  ```
- **문제**: 태그가 없으면 품질 게이트 #6 FAIL → 재생성 또는 Reviser 태그 주입 유도
- **수정 방향**: 체크 로직을 자연어 표지 패턴 검사로 교체  
  ```python
  " 6. 해석·전망 조건부 언어 표지가 적용됐는가?":
      bool(re.search(r"예상된다|전망된다|분석이다|해석된다|가능성이 높다", content))
  ```

### 이슈 #3 (P0): GPT Writer 자기검수 항목이 태그 삽입 지시

- **파일**: `generator.py`
- **위치**: GPT Writer Post1 시스템 프롬프트 자기검수 섹션 (5424행)
- **내용**:
  ```
  □ ★ "2026년", "연간", "가이던스"... 이 포함된 문장이 [전망] 태그 없이 현재 사실로
       단정 서술됐는가 (있으면 [전망] 태그 추가 또는 조건부 해석 문장으로 분리)
  ```
- **문제**: GPT 라이터 본문 규칙은 태그 삽입 금지이나, 자기검수 항목은 태그 추가 지시  
  → GPT가 태그를 삽입하는 경우 발생
- **수정 방향**: `[전망] 태그 추가` 지시 삭제, `조건부 해석 문장으로 분리` 규칙만 유지

### 이슈 #5 (P1): Phase 15D/E/F가 일부 [전망]만 제거 — 전체 제거 로직 부재

- **파일**: `generator.py`
- **위치**: `_strip_misapplied_jeonmang_tags` / `_enforce_current_year_month_settlement` / `_strip_current_year_past_month_jeonmang` (7149~7161행 파이프라인 호출)

- **문제**: 정상적인 미래 전망 문장에 붙은 `[전망]` 및 모든 `[해석]` 태그는 발행까지 잔존
- **수정 방향**: 발행 직전 **모든 `[해석]` / `[전망]` 태그를 일괄 제거**하는 후처리 함수 추가  
  (`publisher.py` 또는 `generator.py`의 최종 반환 직전에 삽입)

### 이슈 #6 (P1): 내부 품질 추적 코드(Phase 13/14I)가 [해석] 태그 기반 의존

- **파일**: `generator.py`

### 이슈 #7 (P2): `_normalize_tag` 함수가 오형성 태그를 교정(보존)

- **파일**: `generator.py`
- **위치**: 4712~4721행 (`_normalize_tag` 함수)
- **내용**:
  ```python
  content = re.sub(r"\[해해석\]",  "[해석]", content)
  content = re.sub(r"\[전전망\]",  "[전망]", content)
  ```
